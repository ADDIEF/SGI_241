ANÁLISIS ESTRUCTURADO

¿QUÉ ES?

El análisis estructurado es como el mapa que usamos antes de construir un sistema. Nos ayuda a dividir el sistema en pedazos más pequeños y entendibles para saber exactamente qué tiene que hacer cada parte. En lugar de preocuparnos por cómo vamos a programar todo, nos enfocamos en entender qué necesita el sistema y cómo los diferentes elementos trabajan juntos para cumplir con esos objetivos.


Cuando desarrollamos un sistema de gestión de inventarios para un supermercado, usamos diversas herramientas para representar visualmente y documentar cómo funcionará el sistema. Estas herramientas nos permiten identificar, analizar y planificar cada parte del sistema de manera clara y organizada.


¿CÓMO LO HACEMOS?

Para planificar el sistema, utilizamos modelos y diagramas que nos ayudan a entender los diferentes aspectos del proyecto y cómo deben interactuar entre sí.


  • Modelo Ambiental:

    Este modelo describe las interacciones externas del sistema y cómo se conecta con su entorno. Nos ayuda a entender los límites del sistema y qué elementos externos (como usuarios o dispositivos) interactúan con él.

  • Modelo de Comportamiento:

    Se enfoca en cómo reacciona el sistema a diferentes estímulos o acciones del usuario. Aquí representamos los procesos internos y las respuestas del sistema ante ciertas condiciones o eventos.

  • Diagrama de Secuencia:

    Este diagrama nos permite visualizar cómo los objetos y actores se comunican entre sí en un orden específico para completar una tarea o proceso dentro del sistema.

  • Casos de Uso:

    Definimos qué acciones puede realizar un usuario dentro del sistema y cómo interactúa con él para cumplir con un objetivo específico, como registrar una venta o generar un reporte.

  • Casos de Uso Extendido:

    Ampliamos los casos de uso principales para detallar escenarios más específicos o excepciones, como qué pasa si un usuario intenta realizar una acción con permisos insuficientes.


¿QUÉ GANAMOS CON ESTO?

Estas herramientas nos brindan varias ventajas que hacen que el desarrollo sea más eficiente y claro:

  • Claridad: Podemos visualizar cómo funcionará cada parte del sistema antes de implementarlo.
  • Organización: Nos permite estructurar el trabajo y delegar tareas específicas dentro del equipo.
  • Detección de problemas: Al mapear el sistema, podemos identificar inconsistencias o errores en una etapa temprana.
  • Flexibilidad: Si el cliente solicita cambios, es más fácil ajustarnos porque ya tenemos una base sólida.

HERRAMIENTAS QUE USAMOS

Las herramientas que empleamos para modelar el sistema son:

  • Modelo Ambiental: Para definir los límites del sistema y sus interacciones externas.
  • Modelo de Comportamiento: Para analizar cómo el sistema responde a eventos y estímulos.
  • Diagrama de Secuencia: Para representar el flujo de comunicación entre actores y objetos.
  • Casos de Uso: Para documentar qué puede hacer cada usuario en el sistema.
  • Casos de Uso Extendido: Para detallar escenarios excepcionales o procesos más complejos.
